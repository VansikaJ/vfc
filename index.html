Architecture

Three layers.
One seamless system.

☁️

Cloud Layer — Centralized Intelligence

Long-term storage, global analytics, model training, policy management. Handles non-time-critical workloads with high computational power.

🌫️

Fog Layer — RSUs & Traffic Signals

Distributed processing nodes at the network edge. Handles real-time V2X data, collision alerts, and latency-critical decisions within 5ms.

🚗

Data origin and computation destination. Cars, pedestrian sensors, IoT devices generating and consuming real-time telemetry data.

Why fog — not cloud?

Traditional cloud architectures centralize data processing in distant datacenters. For vehicles moving at 60mph, a 50ms cloud round-trip is the difference between safe braking and a collision.

Fog computing solves this by bringing compute to the curb — literally. Roadside units and traffic signals become mini-datacenters, processing data within 5ms of generation. The cloud handles the big picture; fog handles the moment.

The architecture comes in two-, three-, and four-layer variants, each adding controller and application planes for larger urban deployments. Four-layer designs offer the highest modularity, supporting heterogeneous communication environments.

Two-layer: RSU + Vehicle (simple, less scalable)
Three-layer: + Controller (urban deployments)
Four-layer: + Application plane (maximum adaptability)

Task Scheduling

Three schools of
thought on task management.

How fog nodes decide which tasks get compute resources — and how fast — is one of the field's most contested design questions. Three dominant paradigms each make distinct trade-offs.

Convex Optimization
CENTRALIZED
Optimality
Real-time speed
Scalability
Uses Lyapunov optimization & ADMM. Mathematically optimal in static/semi-static environments. High computational complexity is the critical limitation for live vehicular networks.
Heuristic Methods
CENTRALIZED
Optimality
Real-time speed
Scalability
EDF, FCFS, genetic algorithms, particle swarm. Fast and easy to deploy. Sacrifices global optimality for speed — well-suited for traffic signal control and collision avoidance timing.
Machine Learning / DRL
ADAPTIVE
Optimality
Real-time speed
Scalability
Deep Reinforcement Learning (DRL), Q-learning. Achieves 20% latency reduction in cooperative driving. Requires significant training data and raises interpretability concerns for safety-critical systems.

Security & Privacy

Every connected car
is an attack surface.

The 2017 Jeep Cherokee hack demonstrated the catastrophic potential of vehicular security breaches — remote control of brakes and steering via the infotainment system. In fog-enabled ITS, the stakes are systemic.

⚠ Active Threat Landscape

Eavesdropping

Unprotected V2X messages intercepted by unauthorized parties. Vehicle locations, speeds, and routes exposed.

Sybil Attacks

Fake vehicle identities flood the network, misleading traffic management systems and causing coordinated chaos.

Man-in-the-Middle

Adversaries intercept and alter traffic data, feeding false information to autonomous vehicle decision systems.

DoS on Fog Nodes

Overwhelms edge processors, crippling real-time collision avoidance and emergency response systems.

Compromised RSUs

Attackers gain control of roadside units — the critical infrastructure backbone of the fog network.

✓ Defense Strategies

Blockchain Authentication

Decentralized tamper-proof identity verification using smart contracts. Reduces unauthorized access without single points of failure.

Post-Quantum Cryptography

Kyber lattice-based encryption (NIST-standardized). Secures V2X comms against future quantum computing threats.

Zero-Trust Architecture

Continuous entity verification minimizes attack surface. No implicit trust — every node must authenticate every time.

Federated Learning

Collaborative threat detection without sharing sensitive data. Intel's FL frameworks improve detection accuracy by 15%.

Pseudonym-Based Auth

Frequent identity rotation prevents linkage attacks while maintaining operational security. Required under GDPR/CCPA.

Future Research

Where the road
leads next.

The field is converging on a set of technologies that, combined with fog computing, promise to deliver a transportation infrastructure that is fully autonomous, quantum-resistant, and sustainably efficient.

5G → 6G Networks
Sub-millisecond vehicular communication
6G promises latency reduction beyond 5G's current ~5ms floor, with network slicing enabling dedicated security channels for autonomous vehicles. The EU's 5G-MOBIX project is the proving ground, already validating cross-border autonomous mobility at scale.
Federated Learning
Privacy-preserving collective intelligence
Vehicles and fog nodes collaboratively train threat detection models without exposing raw data. Intel's edge FL frameworks demonstrate 15% improvement in detection accuracy. The challenge: synchronization in intermittently connected networks.
Quantum-Resistant Cryptography
Hardening V2X against the quantum threat
The Kyber algorithm — a NIST-standardized lattice-based post-quantum system — is positioned to replace current encryption in V2X communications. Balancing computational overhead with vehicular resource constraints remains an open research problem.
Hybrid Scheduling Frameworks
The best of all three paradigms
No single scheduling strategy covers VFC's demands. The proposed direction: lightweight convex optimization for stable scenarios + heuristics for real-time responsiveness + DRL for predictive adaptability. A unified framework that adapts to network state dynamically.
Digital Twin Networks
Safe simulation of unsafe scenarios
Complete virtual replicas of city-scale vehicular networks allowing testing of edge cases — mass incident response, cyberattack simulation, extreme weather — without real-world risk. Expected to become standard infrastructure for ITS validation by 2030.
